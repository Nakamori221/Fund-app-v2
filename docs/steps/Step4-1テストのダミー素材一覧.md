## 2) INT（インタビュー）小ループ：実務の回し方

**狙い**：pending を最小回数の面談で埋める。面談は **数値＋根拠ファイル＋As-of** を必ずセットで回収。

### 2-1. 準備

1. `pending_list` をロール別に割当（CFO／Legal／CRO…）。
    
2. 質問票テンプレ（上記MD）を配布。必要な**CSVテンプレ**も同封。
    
3. 同意取得（録音/記録・利用範囲：IC/LP）。
    

### 2-2. 面談当日（例：NimbusFlow の pending）

- **CFO**
    
    - 粗利率78%の**内訳**（ホスティング・外注・決済手数料など％とUSD）→ 「粗利率内訳CSV」へ記入
        
    - **12ヶ月コホート残存**（分母定義と表）→ 「コホート残存率CSV」へ記入
        
- **Legal/BD**
    
    - **主要ディール条項**（清算優先／反希薄化／情報権／ボード…）→ 「主要ディール条項CSV」へ記入
        

### 2-3. 取込 → 承認（半自動）

1. MCPの **CONFインジェスタ** にCSV/ファイルを投入（GDrive/Notionから取込可）。
    
2. **抽出プレビュー**で、辞書IDにマップされた候補値を確認。
    
3. **差分ビュー**で既存値との衝突を解決（優先：CONF > INT > PUB > EXT）。
    
4. **LPマスキング・プレビュー**で表示確認（IC / LP+NDA / LP）。
    
5. **承認→確定登録**（監査ログに記録）。
    

### 2-4. 自動再計算 → ドキュ再生成

- LTV/CAC・バリュエーションなど **ANL** が再実行。
    
- **IC版／LP版**の両方を再生成（差分サマリに「更新点」を自動記載）。

## 3) 役職別・実務質問セット（要点）

**CFO / FP&A**（CSV同封）

- 粗利率：内訳（％／USD）・期間・為替。
    
- コホート：開始顧客数／新規／解約／期末、MRR内訳（拡張・縮小・解約）。
    
- LTV/CACの**定義**（分母・分子の期間整合）と現在値。
    
- 予実差異（直近6ヶ月）・ランウェイ。
    

**Legal / BD**（CSV同封）

- 清算優先（倍率・参加型/非参加型）
    
- 反希薄化（加重平均／フルラチェット／なし）
    
- 情報権（頻度・対象）／ボード（議席数／オブザーバー）
    
- 主要顧客契約の最低保証・解約条項（**IC限定**で詳細）
    

**CEO / CPO / CRO / CMO**（MD参照）

- 「なぜ今」一次ソース、ロードマップ6ヶ月確定、チャネル別CPA/CVR、解約理由Top3 等。

## 4) CONF 承認UI：動作フロー（ワイヤーの要点）

1. **A：アップロード**（GDrive/Notion/MCP）
    
2. **B：抽出プレビュー**（辞書IDに正規化／開示レベル初期値・機微タグ自動）
    
3. **C：衝突解決**（既存 vs 新規／保留可）
    
4. **D：LPマスク・プレビュー**（IC/LP+NDA/LP 切替）
    
5. **E：確定登録 & 監査ログ**（誰が・何を・どう確定したか）
    

> **ポリシー反映**：Step 3の `/config/lp_masking_policy_v0_1.json` を参照して、自動マスキング（レンジ化／丸め／匿名化）をプレビューします。

- ディール条項は LP で「1x非参加／加重平均」など標準語彙に変換され脚注付き

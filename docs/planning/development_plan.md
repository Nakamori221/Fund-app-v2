## ステップ別補足課題

- Step1（指標定義辞書）：数式・境界値・同義語の変更管理を Git 上で運用するルールが未整備。  
- Step2（収集ルール）：統計値更新頻度と自動再計算ジョブの設計が必要。  
- Step3（マスキング）：disclosure/sensitivity の多段階制御に合わせた権限テーブルと監査ログ設計を同時に実装する必要。  
- Step4（テスト）：source_tag と evidence の突合を自動検証できるツール整備が遅れると検証数が不足。  
- AI/API コストとツール構成：複数文書に分散しており基準値の一本化と差分レビュー手順が必要。  
- ガバナンス：`CONTRIBUTING.md` のレビューフローを Phase1 完了時点で徹底し、レビュー漏れ防止。
